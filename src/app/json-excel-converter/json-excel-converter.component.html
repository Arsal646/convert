Step-by-step guide

Use this quick reference to move between JSON and Excel with confidence.

@if (guideTab() === 'jsonToExcel') {

Convert JSON to Excel

  1. 1 Switch to JSON to Excel mode, then paste or load your JSON array into the editor.
  2. 2 Confirm each object shares the same keys and formatting. The live editor highlights syntax issues instantly.
  3. 3 Press Convert to Excel to generate the preview table and sanity-check the layout.
  4. 4 Download your spreadsheet as Excel or CSV, or go back to adjust the source.

Works well

Flat, consistently keyed objects convert into clean columns:

[
  {"name": "Avery", "role": "Designer", "active": true},
  {"name": "Luca", "role": "Engineer", "active": false}
]

Each property becomes a column, so every row lines up perfectly.

Not recommended

Avoid nested structures or mixed shapes in the same array:

[
  {"name": "Avery", "metrics": {"q1": 12, "q2": 18}},
  {"name": "Luca", "metrics": [12, 18], "active": "yes"}
]

Nested objects and inconsistent types lead to unpredictable columns. Flatten the data first.

} @else {

Convert Excel to JSON

  1. 1 Switch to Excel to JSON, then drag in or browse for a .xlsx or .xls file.
  2. 2 Use the first worksheet with a single header row. Each column header becomes a JSON key.
  3. 3 Review the preview table to confirm blanks, dates, and numbers look correct.
  4. 4 Download the JSON file or copy it directly once you are happy with the structure.

Works well

Keep your spreadsheet tidy before uploading:

  • Row 1 uses simple headers such as Name, Email, Joined.
  • Columns stay consistent - text with text, numbers with numbers.
  • No merged cells, filters, or hidden helper columns.

Not recommended

These patterns create confusing JSON output:

  • Multiple header rows or blank header cells.
  • Merged cells or notes mixed in with data rows.
  • Columns that swap between text, numbers, and dates in the same column.
