Frequently asked questions

Answers to common questions about file limits, formatting, and best practices so you can publish confidently.

Tip

Keep consistent headers across sheets to make JSON output predictable and analytics-ready.

Do I need to clean my JSON before converting?

Yes—make sure each object in the array uses the same keys and uses lowercase or snake_case names. This keeps your spreadsheet columns lined up and helps search engines interpret exported data correctly.

Which Excel formats does the tool support?

Upload .xlsx or .xls files. We read the first worksheet automatically and keep the header row as property names so you can convert back to JSON without manual mapping.

How large can my files be?

The converter comfortably handles a few thousand rows in modern browsers. For very large datasets, split them into logical chunks so you can validate each export and avoid browser memory limits.

Will the converter keep data types intact?

Numbers, booleans, and ISO date strings remain unchanged where Excel supports them. For advanced formatting—such as currency or localized dates—apply formatting inside Excel after export to preserve SEO-friendly raw values.
